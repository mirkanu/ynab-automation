<task type="auto">
  <name>Task 1: Build YnabConnectionSection with PAT input and live budget/account dropdowns</name>
  <files>src/app/(dashboard)/settings/YnabConnectionSection.tsx</files>
  <action>
    Replace the stub (`return null`) with a full `'use client'` component.

    **Component state:**
    - `pat: string` ŌĆö controlled input for the YNAB Personal Access Token
    - `budgets: [{id, name}]` ŌĆö populated after saving PAT
    - `accounts: [{id, name}]` ŌĆö populated after selecting a budget
    - `selectedBudgetId: string`
    - `selectedAccountId: string`
    - `savingPat: boolean`, `savingSelection: boolean`
    - `patSaved: boolean` ŌĆö shows "Token saved" confirmation
    - `error: string`

    **On mount (`useEffect`):**
    - Fetch `/api/ynab/status` to get `{connected, budgetId, accountId}`.
    - If `connected` is true, set `selectedBudgetId` and `selectedAccountId` from the response.
    - Fetch `/api/ynab/budgets` and set `budgets` (only if connected; otherwise wait for user to enter PAT).
    - If `budgetId` is set, also fetch `/api/ynab/budgets/{budgetId}/accounts` and set `accounts`.

    **Save PAT flow (`handleSavePat`):**
    - PUT `/api/settings` with `{YNAB_ACCESS_TOKEN: pat}`.
    - On success: fetch `/api/ynab/budgets` and set `budgets`. Set `patSaved = true`.
    - Show "Token saved ŌĆö select your budget below" message.

    **Budget selection (`handleBudgetChange`):**
    - Set `selectedBudgetId`.
    - Fetch `/api/ynab/budgets/{budgetId}/accounts`, set `accounts`.
    - Clear `selectedAccountId`.

    **Save selection (`handleSaveSelection`):**
    - PUT `/api/settings` with `{YNAB_BUDGET_ID: selectedBudgetId, YNAB_ACCOUNT_ID: selectedAccountId}`.
    - On success: show "Budget and account saved" confirmation.

    **UI layout (match existing Settings card style ŌĆö inline styles, same S object pattern as SenderRulesSection):**
    - Section card with title "YNAB Connection" and description.
    - PAT field: `<input type="password">` with "Save Token" button. Show loading spinner while saving. On success show green "Token saved" text.
    - Budget dropdown: `<select>` populated from `budgets`. Disabled while loading.
    - Account dropdown: `<select>` populated from `accounts`. Disabled while `budgets` loading or `accounts` loading.
    - "Save Budget & Account" button. Disabled during save. Shows "Saving..." while pending.
    - Loading state while fetching on mount: show "Loading YNAB configuration..." text in the section.
    - Error state: red error message below the form.

    **Perceived performance (CLAUDE.md rules):**
    - Show a loading indicator while `/api/ynab/status` loads on mount.
    - Disable buttons during async operations, show spinner/text.
    - Optimistic: after save, show success state immediately (no page reload needed).

    **Props interface:**
    ```typescript
    interface Props {
      onConnectionChange?: (connected: boolean, budgetId: string | null) => void;
    }
    ```
    Call `onConnectionChange` after PAT save and after budget/account save so the parent page can re-render Sender/Currency sections.
  </action>
