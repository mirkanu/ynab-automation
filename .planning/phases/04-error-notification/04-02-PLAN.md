# Plan 04-02 — Wire notifications into webhook handler

## Goal

Add `sendErrorNotification` calls to every error branch in `src/app/api/webhook/route.ts`.

## Error Cases and Routing

| Branch | Recipient | CC |
|--------|-----------|-----|
| Unknown sender | Manuel | — |
| Non-Amazon email (sender known) | Sender | — if Manuel; Manuel if Emily-Kate |
| Claude parse failure | Sender | — if Manuel; Manuel if Emily-Kate |
| YNAB API error (throws) | Sender | — if Manuel; Manuel if Emily-Kate |

Notes:
- "Sender" = the person who forwarded (Manuel or Emily-Kate)
- Emily-Kate errors → To: Emily-Kate + CC: Manuel
- Manuel errors → To: Manuel only
- Non-Amazon: sender must be extracted BEFORE the Amazon check so we have it available

## Implementation Changes to `route.ts`

1. Import `sendErrorNotification` and `MANUEL_EMAIL` from `@/lib/notify`
2. Move `extractOriginalSender` call to BEFORE `isFromAmazon` check
3. In non-Amazon branch: fire notification to sender (CC Manuel if sender is Emily-Kate), then return
4. In unknown-sender branch: fire notification to Manuel only
5. In Claude parse failure branch: fire notification to sender (CC Manuel if Emily-Kate)
6. In YNAB error branch: wrap `createYnabTransaction` in try/catch; on error fire notification then return

  : (process.env.EMILY_KATE_EMAIL ?? '');
