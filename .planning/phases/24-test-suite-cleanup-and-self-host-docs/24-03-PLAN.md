  <name>Task 1: Create Railway deploy template and validate it</name>

  <action>
Create a Railway deploy button template so clicking it produces a deployment with PostgreSQL provisioned, DATABASE_URL set, and IRON_SESSION_SECRET auto-generated.

Railway templates are GitHub repos with a `railway.json` (or `railway.toml`) that describes services, plugins, and env var configuration. The deploy button URL format is:
```
https://railway.com/template/<template-slug>?referralCode=<optional>
```
or using the GitHub template endpoint:
```
https://railway.com/new/template?template=https://github.com/<owner>/<repo>
```

The simplest approach for this project is the `https://railway.com/new/template?template=<repo-url>` format which uses the repo's own `railway.json` as the template spec.

**Create `railway.json`** (in addition to existing `railway.toml` — Railway reads both):

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "npm run db:migrate && npm start",
    "healthcheckPath": "/api/webhook",
    "healthcheckTimeout": 30,
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 3
  },
  "environments": {
    "production": {
      "IRON_SESSION_SECRET": {
        "generator": "secret"
      }
    }
  }
}
```

The `"generator": "secret"` field tells Railway to auto-generate a cryptographically random value for `IRON_SESSION_SECRET` on first deploy — this is Railway's built-in secret generation feature.

PostgreSQL is provisioned by Railway when the deploy button includes `plugins=postgresql` in the query string. The deploy button URL will be:
```
https://railway.com/new/template?template=https://github.com/manuelmhtr/ynab-amazon-automation&plugins=postgresql
```

**Note:** The exact GitHub repo URL should match the actual repo. If the repo is private or the URL is different, the template approach changes. Since the README's deploy button is currently `https://railway.app/new/template/ynab-automation` (a non-existent template), replacing it with the `?template=<github-url>` approach is the correct v6.0 path.

Use the actual URL in the template link.
