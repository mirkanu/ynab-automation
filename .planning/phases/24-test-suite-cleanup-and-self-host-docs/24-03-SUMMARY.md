## Decisions Made

- Deploy button URL: `railway.com/new/template?template=<github-url>&plugins=postgresql` — this approach requires no Railway template dashboard registration; Railway reads the `railway.json` at the repo root directly
- README is wizard-only: no env-var tables, no git clone instructions, no project structure tree — all were v5.0-era artifacts inappropriate for the non-programmer target reader
- `startCommand` in railway.json runs `npm run db:migrate && npm start` so Prisma migrations apply automatically on every Railway deploy without manual steps
- Wizard step 6 documented as an inbound **email address** (not a URL) — consistent with the INBOUND_EMAIL rename landed in 24-02
