Current src/middleware.ts checks for Auth.js cookies (`authjs.session-token` or `__Secure-authjs.session-token`) and redirects to `/auth/signin`. Needs to instead check for the iron-session cookie named `admin_session` and redirect to `/login`.

The existing middleware matcher already excludes the root `/` and `/api/*` routes. The new matcher must ALSO exclude `/login` and `/logout` to avoid infinite redirect loops.

The iron-session cookie is opaque (encrypted) — middleware cannot decrypt it in the Edge Runtime. The correct check is: does the cookie named `admin_session` exist? If yes → allow through. If no → redirect to `/login`. Actual session validation (is it a valid iron-session with isLoggedIn=true) happens in the dashboard layout server component which has full Node.js runtime access.

The admin-session test file (src/lib/admin-session.test.ts) already has a test for the middleware matcher pattern. After this change the matcher in that test should remain valid — the matcher string the test checks is `'/((?!login|logout|api|setup|_next|favicon.ico).*)'` which is what this plan should implement.

    Final file:
    ```typescript
    import { NextResponse } from 'next/server'
    import type { NextRequest } from 'next/server'

    /**
     * Lightweight middleware that checks for iron-session cookie existence only.
     * Actual session validation (isLoggedIn=true) happens in server components
     * via getAdminSession() which has Node.js runtime access.
     *
     * Cannot decrypt the iron-session cookie here — Edge Runtime cannot run
     * Node.js crypto. Cookie presence is sufficient to let the server component
     * perform the real check.
     */
    export function middleware(request: NextRequest) {
      const sessionCookie = request.cookies.get('admin_session')

      if (!sessionCookie) {
        const loginUrl = new URL('/login', request.url)
        loginUrl.searchParams.set(
          'callbackUrl',
          request.nextUrl.pathname + request.nextUrl.search
        )
        return NextResponse.redirect(loginUrl)
      }

      return NextResponse.next()
    }

    export const config = {
      matcher: [
        // Protect page routes only.
        // Excludes: login, logout, setup wizard, API routes, Next.js internals, favicon
        '/((?!login|logout|setup|api|_next/static|_next/image|favicon.ico).*)',
      ],
    }
    ```

    The matcher pattern `/((?!login|logout|setup|api|_next/static|_next/image|favicon.ico).*)` ensures:
    - `/login` → not matched → loads without redirect (public)
    - `/logout` → not matched → API route handles its own auth
    - `/setup` → not matched → Phase 23 wizard (public for first install)
    - `/api/*` → not matched → API routes return 401 themselves
    - `/dashboard`, `/settings`, `/logs`, `/tools` → matched → cookie check applies
