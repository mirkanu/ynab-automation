**Step A — Create the Railway project.**

Use the `railway` CLI to create a new project named `ynab-screenshots`, link the same GitHub repo (`mirkanu/ynab-automation`), and add a PostgreSQL plugin. Commands (run from /data/home/ynab):

```bash
# Create and link a new Railway project
railway init --name ynab-screenshots
# When prompted for source: select "Empty project"
# Note the project ID printed

# Add PostgreSQL plugin to the project
railway add --plugin postgresql
# Wait for Postgres to provision and note the DATABASE_URL

# Link to GitHub repo and deploy
railway up --detach
# This queues a deployment from the current HEAD. Wait for green status.
```

If `railway init` is non-interactive in this environment, use:
```bash
railway init -n ynab-screenshots
```

**Step B — Confirm deployment is live.**

```bash
railway status
railway logs --tail 20
```

Wait until the deployment health check passes (the app starts on `npm run db:migrate && npm start`). The deployment URL will be something like `ynab-screenshots.up.railway.app`.

**Step C — Seed Setting table via psql through Railway TCP proxy.**

Get the Postgres connection details for the NEW project (not ynab-test-production):
```bash
railway variables --json | grep DATABASE
# or
railway connect postgresql
```

Use psql to seed Setting rows. The TCP proxy for the new project will be a different host than `mainline.proxy.rlwy.net:44022`. Get the correct proxy URL from `railway connect postgresql` output, then:

```bash
psql "$NEW_DATABASE_URL" <<'SQL'
INSERT INTO "Setting" (key, value) VALUES
  ('ADMIN_PASSWORD', 'ScreenshotDemo123'),
  ('YNAB_ACCESS_TOKEN', 'screenshot-dummy-token'),
  ('YNAB_BUDGET_ID', 'screenshot-budget'),
  ('YNAB_ACCOUNT_ID', 'screenshot-account'),
  ('ANTHROPIC_API_KEY', 'sk-ant-screenshot'),
  ('RESEND_API_KEY', 're_screenshot'),
  ('INBOUND_EMAIL', 'screenshot@upload.pipedream.net'),
  ('WIZARD_COMPLETE', 'true')
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;
SQL
```

**Step F — Write `scripts/capture-feature-screenshots.ts`.**

Model it on the existing `scripts/capture-wizard-screenshots.ts`. The new script:
- Reads `SCREENSHOT_BASE_URL` env var (required — no localhost default since this targets the Railway deployment)
- Reads `SCREENSHOT_PASSWORD` env var (default: `ScreenshotDemo123`)
- Logs in at `/login` by filling the password field and clicking Submit, waits for redirect to `/dashboard`
- Screenshots three pages at 1280×800, `fullPage: false`:
  1. `/dashboard` → `docs/images/feature-dashboard.png` (wait for: `networkidle`)
  2. `/settings` → `docs/images/feature-settings.png` (wait for: `networkidle`)
  3. `/logs` → `docs/images/feature-activity-log.png` (wait for: `networkidle`, or first table row visible)
- Saves raw PNGs to `docs/images/` (overwriting if they exist)
- Prints the deployment URL and output file paths on completion

Add npm script to package.json:
```json
"docs:feature-screenshots": "npx tsx scripts/capture-feature-screenshots.ts"
```
