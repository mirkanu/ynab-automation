**railway.toml** — Railway deployment config telling Railway how to build and start the app. Railway auto-detects Next.js but explicit config avoids surprises:
```toml
[build]
builder = "NIXPACKS"

[deploy]
startCommand = "npm run db:migrate && npm start"
healthcheckPath = "/api/webhook"
healthcheckTimeout = 30
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 3
```

The `startCommand` runs Prisma migrations on each deploy before starting the server. This ensures the `ProcessedEmail` table exists when the app boots.

Note: `healthcheckPath = "/api/webhook"` will return 405 (Method Not Allowed) on GET, not 200. Railway health checks accept any non-5xx response. If Railway requires a 200, add a GET handler to the webhook route that returns `{ status: 'ok' }`.
