This task deploys the scaffolded app to Railway using the Railway CLI.

**Step 1 — Commit scaffold files from Plan 01:**
```bash
cd /data/home/ynab

git commit -m "feat: scaffold Next.js app with webhook stub and Railway config"
```

**Step 2 — Check if Railway CLI is installed:**
```bash
railway version
```
If not installed: `npm install -g @railway/cli`

**Step 3 — Link to Railway project:**
```bash
cd /data/home/ynab
railway login --browserless
railway link
```
Select the existing Railway project (the one running Josie) or create a new one named `ynab-automation`.

**Step 4 — Deploy:**
```bash
railway up --detach
```
This triggers a Railway build and deploy. The `--detach` flag returns immediately; Railway builds in the background.

**Step 5 — Check deploy status and get domain:**
```bash
railway status
railway logs --tail 50
railway domain
```
Wait for the build to complete. The start command `npm run db:migrate && npm start` runs Prisma migrations automatically, creating the `ProcessedEmail` table. Note the assigned domain (e.g., `ynab-automation-production.up.railway.app`).

**Step 6 — Smoke test the live endpoint:**
```bash
DOMAIN=$(railway domain 2>/dev/null)
curl -s "https://$DOMAIN/api/webhook"
# Expected: {"status":"ok"}

curl -s -X POST "https://$DOMAIN/api/webhook" \
  -H "Content-Type: application/json" \
  -d '{"test": true}'
# Expected: {"received":true}
```

**What was automated in Tasks 1-2:**
- Next.js app deployed to Railway with HTTPS endpoint
- PostgreSQL provisioned and ProcessedEmail table migrated via Prisma
- Placeholder secrets set (DATABASE_URL is real; the other three are placeholders)

**Step 1 — Replace placeholder secrets with real values in Railway dashboard:**

Open Railway dashboard -> your service -> Variables tab. Replace each placeholder:
- `ANTHROPIC_API_KEY` → your Anthropic API key (from https://console.anthropic.com -> API Keys)
- `YNAB_PERSONAL_ACCESS_TOKEN` → your YNAB personal access token (from https://app.ynab.com -> Account Settings -> Developer Settings -> Personal Access Tokens)
- `MAILGUN_API_KEY` → your Mailgun server token (from Mailgun Dashboard -> Your Server -> API Tokens)

Click "Save" — Railway will automatically redeploy with the real values.

**Step 2 — Configure Mailgun inbound webhook:**

In your Mailgun account (https://account.mailgunapp.com):
1. Go to your server -> Settings -> Inbound
2. Enable inbound processing if not already enabled
3. Set the Webhook URL to: `https://{your-railway-domain}/api/webhook`
4. Note the Mailgun inbound email address shown on this page (e.g. `abc123@inbound.mailgunapp.com`) — this is the address Manuel and Emily-Kate will forward Amazon emails to

**Step 3 — Verify the live endpoint from your terminal:**
```bash

# Expected: {"status":"ok"}

curl -s -X POST https://{your-railway-domain}/api/webhook \
  -H "Content-Type: application/json" \
  -d '{"test": true}'
# Expected: {"received":true}
```
