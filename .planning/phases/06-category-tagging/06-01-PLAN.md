  <behavior>
    - extractCategoryHint('') returns null
    - extractCategoryHint(html with no text before blockquote) returns null
    - extractCategoryHint(html where first visible text is "Groceries") returns "Groceries"
    - extractCategoryHint(html where first visible text is "  groceries  " with whitespace) returns "groceries" (trimmed)
    - extractCategoryHint(html where first line is blank and second has text) returns that text (skip blank lines)
    - extractCategoryHint(html that is a raw forwarded email with no user-typed text — blockquote immediately) returns null
    - Text lines inside a blockquote or after the first blockquote are NOT returned
    - extractCategoryHint(html where only pre-blockquote text is a Gmail signature like "Kind regards, Manuel") returns null
    - extractCategoryHint(html where user typed "Groceries" followed by an auto-signature) returns "Groceries" (signature ignored)
    - extractCategoryHint(html where only pre-blockquote text is "-- " separator line) returns null
    - extractCategoryHint(html where only pre-blockquote text is "Sent from my iPhone") returns null
  </behavior>
  <action>
Add `extractCategoryHint(html: string): string | null` to src/lib/email.ts.

Implementation approach:
1. Before any other processing, remove email signature HTML elements from the raw HTML. Gmail wraps signatures in `<div class="gmail_signature"...>...</div>`. Remove these with a regex: `html.replace(/<div[^>]+class="gmail_signature"[^>]*>[\s\S]*?<\/div>/gi, '')`.
2. Find the index of the first `<blockquote` occurrence in the cleaned HTML. Take `html.slice(0, blockquoteIndex)` (if no blockquote found, use full html).
3. Strip all remaining HTML tags from that slice (replace with a space), then collapse multiple spaces/newlines into single newlines.
4. Split on newlines, trim each line.
5. Walk lines top-to-bottom. Stop (return null) if a line matches a signature pattern:
   - `/^--\s*$/` (email sig delimiter)
   - `/^(kind|best|warm|with|many)\s+regards/i`
   - `/^thanks,?$/i`
   - `/^sent from /i`
6. Skip blank lines. Return the first non-empty, non-signature line, or null if none found.

Add tests in src/lib/email.test.ts using the same vitest import pattern as existing tests in that file. Test with realistic HTML snippets that include a `<blockquote>` section representing the forwarded email, and snippets with Gmail signature divs and plain-text signature lines.
  </action>
