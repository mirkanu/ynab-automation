<task type="auto">
  <name>Task 2: Create POST /api/setup/step — wizard step save endpoint</name>
  <files>src/app/api/setup/step/route.ts</files>
  <action>
Create `src/app/api/setup/step/route.ts`. This endpoint is PUBLIC (no auth guard) because the wizard runs before any admin password is set.

Request body: `{ step: number, settings: Record<string, string> }`

Logic:
1. Validate body — `step` must be 1-6, `settings` must be a non-empty object.
2. Call `saveSettings(body.settings)` to persist the values to the Setting table.
3. If `step === 6` (final step), also call `saveSettings({ WIZARD_COMPLETE: 'true' })` to mark the wizard done.
4. Call `deriveWizardStep()` to compute the next step.
5. Return `{ success: true, nextStep: <number> }`.

Error handling: return 400 for invalid body, 500 for DB errors with `{ error: string }`.

Imports: `saveSettings` from `@/lib/settings`, `deriveWizardStep` from `@/lib/wizard`.

Do NOT import `getAdminSession` — this route must be reachable before a password exists. The middleware.ts already excludes `/api` routes from cookie checks, so no additional bypass is needed.
