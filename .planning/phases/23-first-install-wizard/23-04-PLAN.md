**Step A — Capture current Setting rows:**

Use the Railway TCP proxy to connect to the production database and export all Setting rows:

```bash
# Get the DATABASE_URL from Railway (contains TCP proxy credentials)
railway variables --json | grep DATABASE_URL

# Export all Setting rows to a backup file
psql "$DATABASE_URL" -c "SELECT key, value FROM \"Setting\" ORDER BY key;" > /tmp/setting-backup-$(date +%Y%m%d-%H%M%S).txt
```

  <name>Task 5: Restore production Setting table from backup</name>
  <files></files>
  <action>
After human verification is complete (approved or after any fixes are verified), restore the production Setting table from the backup captured in Task 2.

**Step A — Clear the test data written during verification:**
```bash
psql "$DATABASE_URL" -c "DELETE FROM \"Setting\";"
```

**Step B — Restore from the CSV backup:**
```bash
BACKUP_FILE=$(ls /tmp/setting-backup-*.csv | tail -1)
echo "Restoring from: $BACKUP_FILE"

# Read CSV (skip header) and insert each row
tail -n +2 "$BACKUP_FILE" | while IFS=, read -r key value; do
  # Strip surrounding quotes if present
  key=$(echo "$key" | sed 's/^"//;s/"$//')
  value=$(echo "$value" | sed 's/^"//;s/"$//')
  psql "$DATABASE_URL" -c "INSERT INTO \"Setting\" (key, value) VALUES ('$key', '$value') ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;"
done
```

**Step C — Verify restore:**
```bash
psql "$DATABASE_URL" -c "SELECT key, length(value) as value_len FROM \"Setting\" ORDER BY key;"
# All original rows should be present. WIZARD_COMPLETE should be 'true' (from original config).
```
