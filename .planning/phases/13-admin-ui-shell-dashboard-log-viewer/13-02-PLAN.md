<objective>
Build the activity log viewer at /logs with filtering, pagination, and expandable row details.

Purpose: Lets the admin drill into any pipeline run to see exactly what happened — the raw email, Claude's parse output, the YNAB result, and any errors. Essential for debugging and monitoring.

Output: A /logs page with status/date filters, paginated log list, and expandable detail view per entry.
</objective>

The /logs page reads URL search params for filtering:
- `?status=success` — filter by status
- `?from=2026-03-01&to=2026-03-27` — date range
- `?page=2` — pagination

These are passed to getActivityLogs() from 13-01's activity-log-queries.ts.
