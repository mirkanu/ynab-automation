<task type="auto" tdd="true">
  <name>Task 1: AES-256-GCM crypto module</name>
  <files>src/lib/crypto.ts, tests/ynab/encryption.test.ts</files>
  <behavior>
    - encryptToken(plaintext): returns base64-encoded string of format `{nonce_hex}:{ciphertext_hex}:{authTag_hex}`
    - decryptToken(ciphertext): returns original plaintext string
    - encryptToken called twice with same input returns different output (random 12-byte nonce per call)
    - decryptToken(encryptToken(x)) === x for any string x
    - Throws if TOKEN_ENCRYPTION_KEY env var is missing or shorter than 32 bytes
    - Never logs or exposes the plaintext token in error messages
  </behavior>
  <action>
    Use Node.js native `crypto` module (already available — no new dependency needed).

    Implementation in src/lib/crypto.ts:
    - Key: derive 32 bytes from `process.env.TOKEN_ENCRYPTION_KEY` using `crypto.createHash('sha256').update(key).digest()` — deterministic, no salt needed since the key is already high-entropy
    - Nonce: `crypto.randomBytes(12)` per encrypt call (12 bytes = 96 bits, standard for GCM)
    - Encrypt: `crypto.createCipheriv('aes-256-gcm', derivedKey, nonce)` → update + final + getAuthTag
    - Output format: `${nonce.toString('hex')}:${encrypted.toString('hex')}:${authTag.toString('hex')}`
    - Decrypt: split on `:`, createDecipheriv, setAuthTag, update + final → returns plaintext string
    - Export: `export function encryptToken(plaintext: string): string` and `export function decryptToken(ciphertext: string): string`

    DO NOT use libsodium (adds a native dep). Node.js crypto is built-in and sufficient.
    DO NOT log tokens in error messages — throw `new Error('Decryption failed')` without including the ciphertext.

    Test in tests/ynab/encryption.test.ts:
    - Set `process.env.TOKEN_ENCRYPTION_KEY = 'test-key-32-bytes-exactly-padded!'` in beforeAll
    - Test: roundtrip (encryptToken → decryptToken returns original)
    - Test: encrypt same string twice → different ciphertexts
    - Test: decryptToken with tampered authTag throws
    - Test: decryptToken with valid input returns exact original string
  </action>

  </verify>
  <done>All 4 encryption tests pass. src/lib/crypto.ts exports encryptToken and decryptToken. No plaintext token visible in error messages.</done>
