  <files>src/app/api/ynab/budgets/route.ts, src/app/api/ynab/budgets/[budgetId]/accounts/route.ts, src/app/api/ynab/selection/route.ts, tests/ynab/budget-selection.test.ts</files>
  <behavior>
    - GET /api/ynab/budgets: 401 if no session; 400 if user has no YNAB connected; returns { budgets: [{id, name}] }
    - GET /api/ynab/budgets/:budgetId/accounts: 401 if no session; 400 if not connected; returns { accounts: [{id, name}] } — filters out deleted and closed accounts
    - PUT /api/ynab/selection with { budgetId, accountId }: 401 if no session; 400 if missing fields; updates selectedBudgetId and selectedAccountId on User; returns { ok: true }
  </behavior>
  <action>
    Create src/app/api/ynab/budgets/route.ts:
    - auth() check → 401
    - getValidYnabToken(session.user.id) — catches "not connected" error → return 400 { error: 'YNAB account not connected' }
    - GET https://api.youneedabudget.com/v1/budgets with Authorization: Bearer {token}
    - Return { budgets: data.budgets.map(b => ({ id: b.id, name: b.name })) }

    Create src/app/api/ynab/budgets/[budgetId]/accounts/route.ts:
    - auth() check → 401
    - Extract budgetId from params
    - getValidYnabToken → 400 if not connected
    - GET https://api.youneedabudget.com/v1/budgets/{budgetId}/accounts
    - Filter: keep only accounts where !account.deleted && !account.closed
    - Return { accounts: filtered.map(a => ({ id: a.id, name: a.name })) }

    Create src/app/api/ynab/selection/route.ts:
    - auth() check → 401
    - Parse { budgetId, accountId } from request.json()
    - Validate both fields present → 400 if missing
    - prisma.user.update where id = session.user.id, set selectedBudgetId, selectedAccountId
    - Return { ok: true }

    Update tests/ynab/budget-selection.test.ts with real tests:
    - Mock getValidYnabToken and fetch
    - Test: returns budget list from YNAB API
    - Test: returns 400 when user not connected
    - Test: accounts filtered (no deleted/closed)
    - Test: PUT /selection persists budgetId and accountId
  </action>
