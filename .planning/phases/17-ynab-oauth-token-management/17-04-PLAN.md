  <name>Task 1: getValidYnabToken with proactive refresh and concurrency mutex</name>
  <files>src/lib/ynab.ts, tests/ynab/refresh.test.ts</files>
  <behavior>
    - getValidYnabToken(userId): if token not expired → decryptToken and return
    - getValidYnabToken(userId): if token expired or within 5-minute buffer → call YNAB refresh endpoint, update DB, return new access token
    - getValidYnabToken(userId): if lastRefreshAttemptAt is within 30 seconds → re-read from DB instead of calling refresh (concurrent mutex)
    - getValidYnabToken(userId): if oauthToken is null → throw Error('User has not connected YNAB account')
    - After refresh: oauthToken, oauthRefreshToken, oauthExpiresAt, and lastRefreshAttemptAt all updated atomically
  </behavior>
  <action>
    Migrate src/lib/ynab.ts:

    1. Add `getValidYnabToken(userId: string): Promise<string>` implementing Pattern 2 + Pattern 3 from RESEARCH.md:
       - prisma.user.findUnique with select: { oauthToken, oauthRefreshToken, oauthExpiresAt, lastRefreshAttemptAt }
       - If !oauthToken → throw new Error('User has not connected YNAB account')
       - Expiry check: `BigInt(Date.now()) >= (oauthExpiresAt ?? BigInt(0)) - BigInt(5 * 60 * 1000)`
       - If not expired → return decryptToken(oauthToken)
       - Concurrent mutex: if lastRefreshAttemptAt and `Date.now() - lastRefreshAttemptAt.getTime() < 30_000`:
         - Re-fetch user from DB
         - If re-fetched token is not expired → return decryptToken(freshUser.oauthToken)
         - Else fall through to refresh (the mutex holder may have failed)
       - POST to https://app.ynab.com/oauth/token with grant_type=refresh_token
       - On failure: throw Error('YNAB token refresh failed — user must re-authorize')
       - On success: prisma.user.update with encryptToken(access_token), encryptToken(refresh_token), new expiresAt, lastRefreshAttemptAt = new Date()
       - Return tokenData.access_token (the new plaintext token for immediate use)

    2. Update ALL existing YNAB API functions to accept `userId: string` as first parameter and use getValidYnabToken(userId) instead of `process.env.YNAB_PERSONAL_ACCESS_TOKEN`:
       - `createYnabTransaction(userId: string, params: YnabTransactionParams): Promise<string>`
       - `getCategories(userId: string, budgetId: string): Promise<YnabCategory[]>`
       - Any other exported functions that call the YNAB API

    3. Update the refresh test stubs in tests/ynab/refresh.test.ts to implement the real tests using vi.mock:
       - Mock prisma to return test user data
       - Mock fetch to return token exchange responses
       - Test: token not expired → decryptToken called, no fetch call
       - Test: token expired → fetch called with refresh_token grant
       - Test: concurrent mutex → second call within 30s reads from DB, not fetch
       - Test: null oauthToken → throws

    DO NOT remove YNAB_PERSONAL_ACCESS_TOKEN from .env or .env.example yet — callers in webhook/route.ts will be updated in the next step of this task (Task 1 also updates callers, not just ynab.ts).

    Also update CALLERS to pass userId:

    - src/app/api/test-parse/route.ts: getCategories call, add userId from session.user.id

    If any caller uses YNAB_PERSONAL_ACCESS_TOKEN directly (not through ynab.ts), migrate those too.
