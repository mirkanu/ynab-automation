## Goal

Expand automation beyond Amazon to any retailer order confirmation, and let users optionally tag a YNAB category by typing it on the first line of the forwarded email.

- All tests pass; TypeScript compiles clean

### Phase 6: Category Tagging ✅ Code complete

**Goal:** Users can optionally type a YNAB category name on the first line of a forwarded email; the app looks it up, fuzzy-matches it against the YNAB category list, and assigns it to the transaction if found.

**Requirements:** CAT-01, CAT-02, CAT-03, CAT-04

**Success criteria (all implemented, human verify outstanding):**
1. Typing a valid YNAB category name assigns that category to the created transaction ⚠️ pending verify
2. Category matching is case-insensitive and works for partial name matches ✅ unit tested
3. Forwarding with no first-line text creates an uncategorized transaction ✅ unit tested
4. A non-matching category name creates an uncategorized transaction with no error ✅ unit tested

**Plans:**

#### 06-01 — Library Layer
- `extractCategoryHint(html)`: extracts user-typed text before forwarded blockquote; strips Gmail signatures and common signature line patterns
- `getCategories(budgetId)`: fetches flat `YnabCategory[]` from YNAB budget categories API
- `findCategory(categories, hint)`: case-insensitive substring match
- `YnabTransactionParams.categoryId?`: optional; `createYnabTransaction` conditionally includes `category_id` in POST body
- 67 tests pass; TypeScript clean

#### 06-02 — Webhook Wiring
- `extractCategoryHint` called on raw HTML before Claude parse
- After successful parse: if hint present, `getCategories` + `findCategory` resolve to categoryId
- `getCategories` failure is non-fatal (logged, transaction created uncategorized)
- `categoryId` passed to `createYnabTransaction`
- **Human verify OUTSTANDING** — 3-case test (hint+match, no hint, no match) deferred

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Amazon filter removed entirely (not replaced) | Phase 5 goal is unconditional any-retailer support |
| `payeeName` required (not optional) in YnabTransactionParams | Callers always have retailer from parsed.retailer; explicit is better than implicit |
| Euro routing to WISE_EUR_ACCOUNT_ID | EUR-denominated orders would land in wrong (GBP) account without routing |
| Order date from email, not today | More accurate for YNAB budgeting |
| Category hint extracted from pre-blockquote HTML, not post | User types before forwarding; blockquote contains the forwarded email |
| Gmail signature stripping before hint extraction | Auto-signatures would otherwise be returned as the category hint |
| `getCategories` failure non-fatal | Single lookup failure should not block transaction creation |
| CAT-04 silent on no match | Spec explicitly says no notification if typed category not found in YNAB |
