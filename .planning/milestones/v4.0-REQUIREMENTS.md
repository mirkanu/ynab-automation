# Requirements Archive: v4.0 — Admin Backend UI

**Archived:** 2026-03-28
**Status:** All requirements complete

## Deployment Retirement
- [x] **DEPL-01**: Old Railway deployment is decommissioned (service deleted or redirected)
- [x] **DEPL-02**: README and docs updated to reference only the new deployment URL

## Admin Authentication
- [x] **AUTH-01**: `/admin` routes protected by single ADMIN_PASSWORD env var
- [x] **AUTH-02**: Cookie-based session persists after successful login
- [x] **AUTH-03**: Login page at `/admin/login` with password field
- [x] **AUTH-04**: Logout clears session and redirects to login

## Activity Log
- [x] **LOG-01**: Every inbound email written to DB (from address, subject, timestamp, raw body)
- [x] **LOG-02**: Claude parse output stored per entry (retailer, amount, date, currency)
- [x] **LOG-03**: YNAB result stored per entry (success/fail, transaction ID, amount, account)
- [x] **LOG-04**: Errors stored with full detail (type, message)
- [x] **LOG-05**: Log UI with status filter, date range filter, and pagination
- [x] **LOG-06**: Full request trace expandable per log row

## Settings Editor
- [x] **SET-01**: SENDERS routing editable in UI (add/edit/remove email→account mappings)
- [x] **SET-02**: CURRENCY_ACCOUNTS editable in UI (add/edit/remove currency→account mappings)
- [x] **SET-03**: API keys editable in UI (YNAB_API_KEY, RESEND_API_KEY, RAILWAY_API_TOKEN)
- [x] **SET-04**: Other env vars editable (ADMIN_EMAIL, NOTIFICATION_LABEL, etc.)
- [x] **SET-05**: Settings changes applied to Railway env vars via API (reuse wizard mechanism)

## Dashboard
- [x] **DASH-01**: Emails processed this week and success rate displayed
- [x] **DASH-02**: Last transaction summary (retailer, amount, account, timestamp)
- [x] **DASH-03**: Webhook URL displayed prominently with one-click copy

## Test & Replay
- [x] **TEST-01**: Paste email content → preview Claude parse output (no YNAB write)
- [x] **TEST-02**: Replay any logged email through full pipeline (creates real YNAB transaction)
